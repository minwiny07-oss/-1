# 리눅스 명령어 조사 (과제3)
리눅스 명령어 중에서 top, ps, jobs, kill 명령어에 대해 조사한 내용입니다.

## 요약표
| 명령어 | 역할 | 옵션 |
|--------|------|------------|
| top | 시스템 상태와 진행 중인 프로세스를 실시간으로 모니터링 | -d (갱신 간격), -p(특정 PID만 표시) |
| ps  | 현재 실행 중인 프로세스 목록을 출력 | -e(every process), -f(full format), aux(BSD스타일) |
| jobs | 백그라운드 작업 목록 확인 | (옵션 거의 없음) |
| kill | 프로세스 종료 | -9, -15 등 (시그널 번호) |

## 명령어 상세 설명
### top
-  **설명**: 실시간으로 CPU, 메모리 사용량, 프로세스 정보를 확인할 수 있는 명령어
-  **주요 기능**:
     1. CPU 사용률모니터링
     2. 메모리 사용량 확인
     3. 실행 중인 프로세스 목록 확인
![image](https://github.com/user-attachments/assets/9d41b088-65ac-4549-8347-bf38873a5a1a)


### ps
- **설명**: 현재 실행 중인 프로세스를 출력하는 명령어
- **주요 옵션**:
     1. -e 또는 -A : 모든 프로세스 표시
     2. -f : 자세한 포맷 출력
     3. aux : BSD 스타일로 전체 프로세스 정보 표시

### jobs
- **설명**: 현재 셸에서 실행 중인 백그라운드 작업을 확인하는 명령어
- **사용 상황**:
     1. ctrl + Z로 중지한 작업 보기
     2. &로 실행한 백그라운드 작업 확인

### kill
- **설명**: 특정 프로세스를 종료하거나 시그널을 보낼 때 사용하는 명령어
- **자주 사용하는 시그널**:
     1. -15 : 정상 종료 요청
     2. -9 : 강제 종료
